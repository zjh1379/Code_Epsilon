# AI Agent 提示词使用说明

本项目包含三个专门的AI Agent提示词，用于「异界声律·Epsilon」项目的开发协作。

## Agent 列表

### 1. 需求分析师 Agent (`agent_需求分析师.txt`)

**职责**: 与用户探讨项目需求，分析功能需求，生成详细的开发提示词

**使用场景**:
- 项目需求不明确时
- 需要细化功能规格时
- 需要技术方案设计时
- 需要生成开发任务描述时

**使用方法**:
1. 在Cursor中打开新的Agent Tab
2. 将 `agent_需求分析师.txt` 的内容复制到Agent的系统提示词中
3. 开始与Agent讨论你的项目需求

### 2. 全栈工程师 Agent (`agent_全栈工程师.txt`)

**职责**: 根据需求分析师的提示词，实现完整的前后端代码

**使用场景**:
- 需要实现具体功能代码时
- 需要集成API服务时
- 需要优化代码架构时
- 需要修复bug时

**使用方法**:
1. 在Cursor中打开新的Agent Tab
2. 将 `agent_全栈工程师.txt` 的内容复制到Agent的系统提示词中
3. 将需求分析师生成的开发提示词提供给该Agent
4. Agent将根据提示词实现代码

### 3. 角色设定专家 Agent (`agent_角色设定专家.txt`)

**职责**: 创建和优化虚拟角色的设定，生成LLM的system prompt，确保角色生动有趣

**使用场景**:
- 需要创建新的虚拟角色时
- 需要优化现有角色的设定时
- 需要调整角色的语言风格时
- 需要生成角色的system prompt时
- 需要让角色更加生动自然时

**使用方法**:
1. 在Cursor中打开新的Agent Tab
2. 将 `agent_角色设定专家.txt` 的内容复制到Agent的系统提示词中
3. 与Agent讨论你想要的角色特征和风格
4. Agent会生成完整的角色设定和system prompt
5. 可以根据对话效果继续优化角色设定

## 工作流程

### 推荐流程

1. **角色设定阶段**（可选）
   - 使用「角色设定专家 Agent」创建或优化虚拟角色
   - Agent会生成角色的完整设定和system prompt
   - 可以多次调整直到满意

2. **需求讨论阶段**
   - 使用「需求分析师 Agent」与用户讨论项目需求
   - Agent会帮助澄清需求，设计技术方案
   - Agent生成结构化的开发提示词

3. **开发实现阶段**
   - 将开发提示词提供给「全栈工程师 Agent」
   - 如果已创建角色设定，将system prompt也提供给开发Agent
   - Agent根据提示词实现代码
   - 可以随时回到需求分析师或角色设定专家处调整

### 独立使用

三个Agent也可以独立使用：
- **角色设定专家**: 专门用于创建和优化角色设定
- **需求分析师**: 专门用于讨论项目需求和技术方案
- **全栈工程师**: 专门用于实现代码功能

### 协作使用

三个Agent可以协作工作：
- 角色设定专家创建角色设定 → 需求分析师设计功能需求 → 全栈工程师实现代码
- 全栈工程师实现过程中发现问题 → 回到需求分析师调整需求
- 角色对话效果不理想 → 回到角色设定专家优化设定

## 项目背景

「异界声律·Epsilon」是一个将LLM文本对话与GPT-SoVITS语音合成技术结合的应用。

**核心功能**:
- Web界面的LLM文本聊天
- 集成GPT-SoVITS实现文本转语音
- 前端显示文本并播放语音

**技术栈**:
- 后端: Python + FastAPI + LangChain
- 前端: React + TypeScript
- TTS服务: GPT-SoVITS API

## 注意事项

- 三个Agent的提示词都是纯文本格式，可以直接复制使用
- Agent会遵循提示词中的开发规范（英文代码，中文解释）
- 不要添加用户要求以外的功能
- 测试性文件使用后要删除
- 角色设定专家生成的system prompt可以直接用于LLM配置

## Agent协作示例

### 完整工作流示例

1. **创建角色**
   ```
   用户 → 角色设定专家: "我想创建一个温柔知性的角色"
   角色设定专家 → 用户: [生成完整的角色设定和system prompt]
   ```

2. **设计功能**
   ```
   用户 → 需求分析师: "需要实现聊天功能，集成TTS"
   需求分析师 → 用户: [生成开发提示词]
   ```

3. **实现代码**
   ```
   用户 → 全栈工程师: [提供开发提示词和角色system prompt]
   全栈工程师 → 用户: [实现代码]
   ```

4. **优化角色**
   ```
   用户 → 角色设定专家: "角色对话太生硬了"
   角色设定专家 → 用户: [优化角色设定，更新system prompt]
   ```
